# Submission Checklist

Use this checklist before submitting the COS 202 Smart Library Circulation & Automation System project.

## PDF Report

Include these sections in the final PDF:

- Project title and course information
- Short system description
- Main features
- Data structures used and where they appear
- Algorithms chosen and why
- UML/class hierarchy diagram
- GUI screenshots for the four main tabs:
  - View Items
  - Borrow / Return
  - Admin
  - Search & Sort
- Challenges faced
- GitHub repository link for the full source code

Suggested source-code note for the PDF:

```text
The full source code is available in the GitHub repository linked below.
The repository contains the src folder, GUI screenshots, class hierarchy diagram,
project report, and run instructions.
```

## GitHub Repository

The GitHub repository should include:

- `src/` source code folder
- `README.md`
- `REPORT.md`
- `CLASS_HIERARCHY.md`
- `SUBMISSION_CHECKLIST.md`
- `gui-screenshots/`
- `run.sh`
- `tools/GuiScreenshotRunner.java`

The repository should not need compiled `.class` files because the app can be compiled from source.

## Optional ZIP

If the submission portal accepts more than one file, submit:

- Final PDF report
- ZIP file of the project source folder

If the portal accepts only a PDF, include the GitHub repository link inside the PDF.
